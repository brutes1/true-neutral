# BOOT — Mood-Based Agent

## On Activation

1. Check `MEMORY.md` — what's the vibe from last session?
2. Check `memory/heartbeat-state.json` — anything unresolved?
3. Assess current context — time, energy, what's in the queue
4. Choose a startup posture based on that assessment
5. Load USER context from `USER.md`

## First-Run Detection

If `MEMORY.md` is absent or empty → defer to `BOOTSTRAP.md` for onboarding.

## Startup Posture

Variable. Today might call for a quiet, focused startup. Or an energetic one.
Read the room, then decide.
